Nazwozbiór

Dostępność

Nazwozbiór został zaprojektowany z myślą o różnorodnych potrzebach użytkowniczek i użytkowników. Poniżej lista wdrożonych rozwiązań.

  • Pomijanie nawigacji (skip-link)

    Pierwszym klikalnym elementem na stronie jest niewidoczny link „Przejdź do treści", który po otrzymaniu fokusu staje się widoczny. Pozwala ominąć blok filtrów i od razu przejść do tabeli z wynikami.

    HTML
  • Widoczny fokus

    Niestandardowe style :focus-visible na wszystkich interaktywnych elementach: przyciskach, linkach, polach formularza. Gruba obwódka w kolorze akcentu (3px) z odstępem (2px).

    CSS UX
  • Obszar live ARIA

    Licznik wyników (span[aria-live="polite"][aria-atomic="true"]) informuje czytniki ekranu o zmianie liczby imion bez przerywania bieżącej czynności.

    ARIA
  • Odpowiednia hierarchia nagłówków

    Strona ma jeden nagłówek <h1> oraz <h2> dla sekcji filtrów. Tabela ma podpis (<caption>) opisujący jej zawartość.

    HTML
  • Kontrast kolorów WCAG AA

    Każda para kolorów (tło/tekst) spełnia współczynnik kontrastu ≥ 4,5:1 dla tekstu podstawowego i ≥ 3:1 dla dużego tekstu, w obu motywach (ciemny i jasny).

    WCAG AA
  • Szacunek dla redukcji ruchu

    Gdy system operacyjny ma włączoną opcję ograniczania animacji (prefers-reduced-motion: reduce), cząsteczki na tle są wyłączone, a wszystkie przejścia CSS (transition) wygaszone.

    CSS
  • Nawigacja klawiaturą

    Wszystkie funkcje są dostępne z klawiatury: wybór płci (Tab + Enter/Spacja), zakresy, lista rozwijana pochodzenia, sortowanie tabeli, rozwijanie szczegółów, losowanie, eksport CSV.

    UX
  • Język strony

    Deklaracja lang="pl" na elemencie <html> informuje czytniki ekranu o języku polskim. Czytelnicy dostają odpowiedni profil wymowy.

    HTML
  • Etykiety ARIA

    Pola formularzy, przyciski i kontrolki mają opisowe etykiety (aria-label lub <label>), np. „minimalna liczba wystąpień", „wylosuj imię z obecnych filtrów".

    ARIA
  • Semantyczny HTML

    Użyto natywnych elementów: <button> zamiast <div onclick>, <table> dla danych tabelarycznych, <select> z <option> dla listy pochodzenia.

    HTML
  • Brak ciasteczek i śledzenia

    Strona nie używa ciasteczek, nie ładuje zewnętrznych skryptów (poza czcionkami z Google Fonts), nie śledzi użytkownika. Prywatność jest dostępnością.

    UX

Nazwozbiór jest projektem hobbystycznym. Deklaracja dostępności powstała w czerwcu 2026 r. Jeśli napotkasz barierę, daj znać na GitHubie.
